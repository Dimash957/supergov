## 🚨 Common Issues & Quick Fixes

### ❌ Ошибка: "Database connection failed"
```bash
# Check .env variables
cat .env | grep SUPABASE

# Should show:
# SUPABASE_URL=https://xxx.supabase.co
# SUPABASE_SERVICE_KEY=eyJ...
```
**Если не видно:** Добавьте эти значения в `.env` и перезагрузите сервер.

---

### ❌ Ошибка: "Missing Stack Auth configuration"
```bash
# Add to .env:
cat << 'EOF' >> .env
STACK_AUTH_JWKS_URL=https://api.stack-auth.com/api/v1/projects/YOUR_PROJECT_ID/jwks
STACK_PROJECT_ID=YOUR_PROJECT_ID
EOF
```
**Получите `YOUR_PROJECT_ID` из Stack Auth панели.**

---

### ❌ Ошибка: "users table not found"
```bash
# Check if table exists
python -c "
from app.database import get_db
db = get_db()
result = db.query('SELECT 1 FROM users LIMIT 1').execute()
print('✅ Table exists')
"
```
**Если ошибка:** Создайте таблицу в Supabase SQL Editor:
```sql
CREATE TABLE IF NOT EXISTS users (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    stack_user_id TEXT UNIQUE NOT NULL,
    email TEXT NOT NULL,
    full_name TEXT,
    phone TEXT DEFAULT '',
    iin TEXT DEFAULT '',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_users_stack_id ON users(stack_user_id);
```

## 📊 Workflow: Debugging Last 30 Minutes

### For each action on app:
1. Check logs in terminal
2. Look for these markers:
   - `🔐 get_current_user() CALLED` ← Auth check started
   - `👤 New user detected` ← Creating profile
   - `✅ User created successfully` ← Profile created
   - `❌ Failed to create user` ← Problem!

### If you see `❌`:
1. Run `python advanced_diagnostics.py`
2. Check Supabase dashboard directly
3. Run `python check_profile_creation.py` again
4. Restart server with: `python -m uvicorn app.main:app --reload --port 8000`

## 🎯 Success Indicators

✅ All good when:
- `python verify_all.py` shows all ✅
- `monitor_login.py` shows `✅ User created successfully`
- Supabase shows new user appeared in `users` table
- Front-end shows profile without errors
- Can access `/api/profile/me` endpoint

❌ Problem when:
- `python verify_all.py` shows ❌ for database
- `monitor_login.py` shows `❌ Failed to create user`
- Supabase `users` table is empty
- Front-end shows "Профиль не найден"
